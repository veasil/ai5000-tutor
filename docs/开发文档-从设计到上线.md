## 2. 技术框架

### 2.1 架构总览
```
[浏览器/孩子端]──┐
[家长端]────────┼─▶ Next.js 15 App Router (Server Actions + Route Handlers)
[运营后台]──────┘        │
                         ├─▶ Prisma ──▶ PostgreSQL 16 (Zeabur)
                         ├─▶ lib/ai/client.ts ──▶ Anthropic SDK (流式)
                         ├─▶ Redis (Zeabur) ──▶ 对话 session 缓存
                         ├─▶ BullMQ ──▶ 审核异步队列
                         └─▶ Sentry (前后端)
```

### 2.2 技术栈（与 `CLAUDE.md` 对齐）
| 层 | 技术 |
|----|------|
| 框架 | Next.js 15 App Router（Server Actions 驱动状态机） |
| DB | PostgreSQL 16 + Prisma |
| 认证 | NextAuth v5（匿名 child + 家长账号） |
| AI | Anthropic SDK，统一封装 `lib/ai/`（**禁止他处直接 new Anthropic / fetch**） |
| 校验 | Zod（前后端共用，见 `types/index.ts`） |
| 样式 | Tailwind CSS + shadcn/ui（用 §1.2 令牌生成 theme） |
| 缓存/队列 | Redis + BullMQ |
| 监控 | Sentry（第一天接入） |

### 2.3 必须遵守的关键约定（来自 `CLAUDE.md`）
1. **AI 调用全部走 `lib/ai/client.ts`**（统一 token 计数/限流/重试/流式格式）。
2. **Prompt 分层**：`system.ts`（小伍人格）→ `stages/level-N.ts` → `guardrails.ts` → 拼最近 20 条对话。
3. **结构化输出**：每轮 AI 同时返回 `text` + `structuredOutput`(JSON)，后者直接写 Journey 字段。
4. **红灯机制**：`lib/ai/safety.ts` 对用户输入与 AI 输出**双向**检查；触发即中断正常流程、写 `journey.safetyFlags`、返回安全文案；自伤/威胁类直接跳出并显示求助资源。
5. **状态机不可跳关**：`state-machine.ts` 校验前置关卡；第 9 关（安全检查）是发布前强制节点，过了才能 `draft → reviewing`。
6. **匿名 ID**：`小伍创客[4位数字]`，不收集真实姓名/学校/地址/精确定位。
7. **Server Action 统一返回** `{ok:true,data} | {ok:false,error,code?}`。
8. **时间戳**用 `DateTime`，禁毫秒时间戳。

### 2.4 设计 → 代码落地方式
- 把 `styles.css` 的 `:root` 令牌转成 Tailwind v4 `@theme`（颜色/字体/阴影/圆角），组件用 shadcn 二次封装为 `Bubble` / `Card` / `LevelTile` / `GuideBand` / `StateView`。
- 静态四屏作为对应路由的视觉验收基线（像素级对照）。

| M3 AI Tutor | `lib/ai/client.ts` 流式；prompt 分层；`/api/tutor/[journeyId]` 返回 text+structuredOutput；前端流式气泡 UI | M2 | P0 |
| M4 安全红灯 | `safety.ts` 双向检查接入 M3；红灯态 UI（见 `states.html` 安全卡）；求助资源页 | M3 | P0 |

| M8 家长端 | dashboard：孩子进度、红灯记录、发布确认 | M1,M2 | P1 |

## 4. 排期（建议 8 周到 MVP，2 人前端 + 2 人后端 + 0.5 设计）

> 以「孩子能完整闯关并安全发布一个作品」为 MVP 终点。按周/Sprint 划分，里程碑为验收点。

| 周 | 阶段 | 主要交付 | 里程碑 |
|----|------|----------|--------|
| W1 | 基建 (M0) | 脚手架、令牌→Tailwind theme、Prisma migration、Sentry、Zeabur 三件套(web/pg/redis)、CI | **M0：本地+预览环境跑通** |
| W2 | 认证+状态机 (M1,M2) | 匿名/家长登录、Journey CRUD、不可跳关校验、Zod schema 共用 | **M1：能创建旅程并保存关卡字段** |
| W3-4 | AI Tutor (M3) | client.ts 流式封装、prompt 分层、tutor 端点、流式对话 UI + 结构化输出写库 | **M2：第 1 关能和小伍真实对话并产出字段** |
| W4 | 安全红灯 (M4) | safety.ts 双向接入、红灯 UI、求助资源（与 M3 并行收尾） | **M3：红灯能拦截并记录 safetyFlags** |
| W5 | 闯关 UI (M5) | 首页/关卡页/地图/徽章 接数据；1-4 关可玩 | **M4：1→4 关连贯可玩** |
| W6 | 作品+审核 (M6,M7) | Demo 生成、作品提交、第 9 关强制检查、家长确认、运营审核队列 | **M5：能提交并通过审核发布** |
| W7 | 家长端+后台 (M8,M9) | 家长 dashboard、运营审核台、风险卡/Demo 模板管理 | **M6：家长/运营可用** |
| W8 | 完善+硬化 (M10) | 空/加载/出错态、移动端走查、无障碍、**MVP 上线安全自检**、压测、灰度 | **M7：MVP 可灰度上线** |

**关键路径**：M0 → M1 → M2 → M3 →（M4 并行）→ M5 → M6/M7。M3（AI Tutor 流式 + 结构化输出）是最大不确定项，建议 W3 先做一关打通端到端再铺量。

**缓冲**：每 2 周留 0.5 天联调缓冲；W8 安全自检若发现 P0 风险（越权/UGC 审核/密钥泄露）优先于新功能。

## 5. 上线前安全自检（W8 必过）
对照《MVP 上线安全自检宝典》重点域：认证与越权、UGC（孩子内容+作品墙）审核、文件/图片上传、API 限流防刷、注入、**AI 应用安全（prompt 注入 / 红灯绕过）**、密钥（`ANTHROPIC_API_KEY` 绝不入前端/仓库）、儿童隐私合规（不收集真实身份）。可用项目内 `mvp-security-check` 流程逐条过。
